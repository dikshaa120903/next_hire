# CareerBoost AI Dashboard - Detailed Project Documentation

## 1. Project Overview
CareerBoost AI Dashboard is a comprehensive, full-stack platform designed to analyze resumes, score them against specific job roles, provide AI-powered refinement suggestions, and recommend matched job opportunities. 

## 2. System Architecture & Tech Stack
The platform follows a standard decoupled architecture featuring a React-based Frontend and a Django-based Backend.

- **Frontend Environment**: React 18, Vite, TypeScript, Tailwind CSS, React Query, Framer Motion
- **Backend Environment**: Django / DRF (Django Rest Framework), Scikit-learn, OpenAI API, SQLite
- **Machine Learning & NLP**: Custom feature extraction, random forest prediction models, local FAISS vector search, and embedding utilities.

## 3. Frontend Features & Details
Located in `frontend/src/`. This layer provides an intuitive and responsive user experience. 

### Core Pages (`frontend/src/pages/`)
- **`Index.tsx`**: The main landing page outlining the platform's benefits and features.
- **`UploadResume.tsx`**: Interface handling user file uploads for resumes, tracking upload states and interacting with the backend ingestion endpoint.
- **`Dashboard.tsx`**: The control center where users view their parsed resume details. It visually displays the overall resume score, the automatically detected target career role, and key extracted metrics (e.g., skill counts, project counts, years of experience).
- **`AISuggestions.tsx`**: A dedicated section to show personalized improvement tips generated by AI (via OpenAI). Offers actionable feedback on formatting, missing skills, or phrasing.
- **`JobRecommendations.tsx`**: Presents a list of curated job opportunities that match the user’s resume profile, driven by the backend ML models.
- **`Login.tsx` & `Signup.tsx`**: Secure authentication views allowing users to create accounts and authenticate securely.
- **`NotFound.tsx`**: Standard fallback routing page.

### Supporting Architecture
- **`lib/`**: Contains utility functions such as `api.ts` which acts as the core interceptor and client for fetching data from the Django endpoints.
- **`components/`**: Reusable UI units (buttons, layout wrappers, cards) driving a consistent design system via Tailwind.

## 4. Backend Modules & Capabilities
Located in `backend/`. The Python backend is split into multiple decoupled Django applications (`apps/`) and a machine-learning utility folder (`utils/`).

### Django Applications (`backend/apps/`)
- **`users/`**: Manages custom user authentication, registration, token generation, and profile management endpoints.
- **`resume/`**: Exposes the `POST /api/upload-resume/` endpoint. Manages the core operations for saving the resume file, extracting raw text, and linking it to the user.
- **`scoring/`**: Handles the robust **Role-Based Scoring System**. Instead of generic scoring, it uses role-specific keywords (e.g., Software Engineering, Data Science, Product Management) to identify a resume's dominant role and compute a fair score out of 100 based on exact skill sets.
- **`ai/`**: Connects to advanced Language Models (OpenAI API). Through `POST /api/improve-resume/`, it pushes parsed resume metrics to generate smart formatting and contextual refinement tips.
- **`recommendation/`**: Manages the `POST /api/recommend-jobs/` endpoint. Compares resume profiles with active job postings using structured similarity search to provide tailored roles.

### ML Utilities (`backend/utils/`)
The `utils` directory handles all heavy computational logic before reaching the Django views.
- **`feature_extractor.py`**: A vital NLP script that parses resumes for particular metrics (skill counts, project counts, years of experience) and compares the findings against pre-defined role-based keyword sets to calculate matches.
- **`model_loader.py`**: Responsible for initializing and interacting with the pre-trained RandomForest model used to predict overall resume scores.
- **`faiss_helper.py` & `embedding.py`**: Integrates Facebook AI Similarity Search (FAISS). This builds dense vector indexes mapping resume text embeddings to job text embeddings for exceptionally fast recommendation matching.
- **`nlp.py`**: Supports base Natural Language Processing functions, likely used for text tokenization, stop-word removal, and cleaning text for vectorization.

## 5. Main Workflow Execution
1. **User Management**: A user registers or logs in via the `Login` page.
2. **Data Ingestion**: User uploads a resume in `UploadResume`. The file goes to the `resume` app which parses the text.
3. **Role & Feature Extraction**: The `feature_extractor.py` analyzes the text. It detects keywords from 11 distinct domains (e.g., Software Engineering, Sales, HR) to pinpoint the candidate’s closest match, calculates metrics, and grades the resume based on that targeted role's baseline.
4. **Scoring Prediction**: The `model_loader.py` leverages the extracted numerical matrices to produce a score from 0-100.
5. **Insights Dissemination**:
   - The user visits the `Dashboard` to review their score and detected role.
   - The user navigates to `AISuggestions` where OpenAI integrations analyze their profile shortfalls to suggest improvements.
   - The user navigates to `JobRecommendations` where `faiss_helper.py` identifies jobs mimicking their embedding profile.
